博文的发布与博客主题替换

Posted on 2020-07-04 Edited on 2020-07-05 In 博客

〇. 前言

距离搭建 Hexo 博客平台，已经过去了一段时间，虽然有在有道云笔记上用 Markdown 把详细过程记录下来，但是由于不可抗力（懒）一直没有把文章上传到博客上面。而且主题也没有换，今天就写篇博客记录一下我发布博客和更换主题中遇到的问题。

一. 博文的发布

2. 问题解决

本地预览时我发现出现了一些问题：

Markdown 文件名成为了博客名

Markdown 中用引用语法写的部分格式错误且图片显示不了

下面依次来解决这些问题。

（1）文件名和博客名的问题

经过搜索了解到这是由于 Markdown 文件一般格式中，会有这样一个部分：

把这一部分内容加上就可以了，但是你用有道云笔记写的时候，这一段是不会默认生成的，需要你手动添加并修改。同时，在有道云笔记预览中，这一段也会变得很丑…

（2）图片显示问题

由于我第一篇博客的图片插入都是用的网络图片插入的方式，且我这个方法是网上随便找到的，极其不靠谱，所以出现了这样的问题。（具体来说就是在有道云笔记中创建一个新笔记，然后向里面放本地图片，然后用的那个图片链接…)

网络图片插入的方式下篇博客尝试一下吧，这里选择的解决办法是用相对路径本地引用的方式。参考这篇文章。

首先，到 blog 目录下查看 _config.yml 文件,查找 post_asset_folder 字段确定 post_asset_folder 设置为 true 。

如图所示

如果用户权限不够的话，就右键 blog 文件夹，在安全里将当前用户的修改权限勾上即可。

设置 post_asset_folder 参数后，在建立文件时，Hexo 会自动建立一个与文章同名的文件夹，您可以把与该文章相关的所有资源都放到此文件夹内，这样就可以更方便的使用资源。

然后把想要插入的本地图片放在同名的文件夹内，插入时路径就写 文件夹名 / 图片名.图片格式 即可。

PS1：偶然间发现，图片名的第一位如果是下划线（即”_”）的话会无法引用。而且图片名中如果有空格也会使语法失效。

PS2:如果没有自动生成同名的文件夹，你手动创建一个同名文件夹也可以。

二. 主题更换

选择了比较热门而且比较简约的 NexT 主题，下面是具体过程的展示。

1. 主题下载

进入 Hexo 官网的主题页面,下拉找到 NexT 主题，点击标题后跳转到项目的 Github 页面。

在 Github 页面下的 README.md 中的 Installation 部分介绍了如何下载主题。

2. 修改配置文件更换主题

修改 blog 目录下的 _config.yml 文件中的 theme 字段，将其设置为 next 后，主题就更换完了。

虽然是疫情期间，但是对于学习来说却是一段好时光。各种折腾之后，总算是大体学会了如何写博客了，接下来要做的是努力坚持更新！
